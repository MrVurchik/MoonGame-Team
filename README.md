## Опис 
Мінімальний прототип: реалізовано двері, які відкриваються при дотику гравця та автоматично закриваються.

## Тести:
Тест 1- Тест симуляції дотику (simulateTouch)
- Перевіряє, що коли гравець торкається дверей, функція повертає true.
- Це означає: логіка спрацювала, двері відкрилися й закрилися.
- Головна мета: підтвердити, що вся послідовність подій працює без помилок.
Тест 2 -Тест зміни станів (isOpen та isAnimating)
- Після симуляції перевіряється, що:
- isOpen в кінці знову false (двері закрилися).
- isAnimating теж false (анімація завершена).
- Головна мета: гарантувати, що змінні стану правильно оновлюються і не залишаються “завислими”.
Тест 3 -Тест позиції дверей (door.Position)
- Після виконання анімації перевіряється нова позиція дверей.
- Це показує, що функція tweenRotation реально змінює CFrame і двері рухаються.
- Головна мета: підтвердити, що математика повороту працює коректно.
